## 1) Conversational assistants (general-purpose)

**Best for:** Open-ended Q&A, brainstorming, drafting, revision, and iterative problem-solving.

**Starter prompt template:**

"I need help with **[goal]**. Context: **[background]**. Constraints: **[time/format/audience]**. Give me a first draft, then ask 3 follow-up questions to improve it."

**Caveat:** Output quality is highly context-dependent; shallow prompts often produce generic answers.

## 2) Voice assistants (embedded conversational)

**Best for:** Fast commands, reminders, timers, hands-free queries, and low-friction utility tasks.

**Starter prompt template:**

"Set **[task]** for **[time/place]**, then remind me again **[backup reminder]**."

**Caveat:** Great for speed and convenience, but less reliable for deep reasoning and long-form composition.

## 3) Generative image/video tools

**Best for:** Rapid concept exploration, style variation, storyboard ideas, mockups, and creative direction.

**Starter prompt template:**

"Create a **[asset type]** in **[style]** showing **[subject]** with **[composition details]**, for **[audience/use case]**."

**Caveat:** Precise results usually require multiple iterations and very specific constraints.

## 4) Hybrid tools spanning all three types

**Best for:** End-to-end workflows where one system can recommend options, discuss tradeoffs, and generate final outputs.

**Starter workflow prompt template:**

"First rank my best options for **[goal]**. Then ask me clarifying questions. Finally generate a final **[artifact]** in **[format]**."

**Caveat:** Convenience is high, but specialized tools may still outperform all-in-one tools for niche tasks.

## 5) How to choose this week (quick rubric)

- If you need **prioritization or ranking**, start with predictive behavior.
- If you need **clarity through dialogue**, start with conversational behavior.
- If you need **new assets or drafts**, start with generative behavior.
- If your workflow has all three stages, use a hybrid flow (rank → discuss → generate).

When in doubt, define your desired output first, then pick the mode that matches that output.
